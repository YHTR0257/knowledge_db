# Key Points

- OKR (Objectives and Key Results) の進捗を評価し、次のサイクルへの学びを得るためのフレームワーク。
- 単なる達成度のスコアリングだけでなく、プロセスからの学び（成功・失敗要因の分析）を重視する。
- 人事評価とは切り離し、挑戦的な目標設定（ストレ-ッチゴール）を促す文化の醸成が鍵となる。

# Context
- 目標達成に向けた組織・個人のパフォーマンスを最大化する必要がある場面で利用する。
- 単なる結果の追跡ではなく、継続的な改善と学習のサイクル（PDCA）を確立することを目的とする。
- 組織全体の透明性を確保し、チームや部門間のアライメント（方向性の一致）を高めるために重要である。

# Body
OKR振り返りは、設定した目標に対する進捗を評価し、組織的な学習を促進するための体系的なプロセスです。主に期末（四半期末など）に実施されますが、週次のチェックインなど、より短いサイクルでの対話も含まれます。

## OKR振り返りの主要ステップ

1.  **スコアリング (Scoring)**
    - 各Key Result (KR) の達成度を、一般的に0.0〜1.0のスケールで客観的に評価します。
    - **60-70% (0.6-0.7) の達成**が「野心的で良い目標設定（ストレッチゴール）」であったと見なされることが多いです。100%達成が続く場合、目標設定が保守的すぎる可能性があります。
    - 定量的なデータや事実に基づいて評価することが重要です。

2.  **成果の共有 (Celebration & Sharing Success)**
    - スコアリングの結果を共有し、達成できたことをチーム全体で認識・祝福します。
    - 目標達成に貢献した具体的な行動や、プロセスの中で生まれたポジティブな副産物（新しい知見、チームワークの向上など）を共有します。

3.  **課題と学びの分析 (Analysis & Learning)**
    - **なぜその結果になったのか**を深掘りします。
    - **成功要因**: 何がうまくいったのか？ 再現性のある要素は何か？
    - **阻害要因**: 何が障害となったのか？ 予期せぬ問題はあったか？
    - 当初の仮説や計画は正しかったか？
    - この分析が、次のサイクルに向けた最も重要なインプットとなります。

4.  **次期アクションへの反映 (Action Planning)**
    - 分析から得られた学びを、次のアクションに具体的に落とし込みます。
    - **継続すること (Keep)**: 成功要因を次期も活かすためのアクション。
    - **改善すること (Improve)**: 阻害要因を排除または軽減するためのアクション。
    - **中止すること (Stop)**: 効果がなかった、あるいは非効率だった活動。
    - これらのインプットは、次期のOKR設定において重要な役割を果たします。

## 実施のポイント

- **CFR (Conversation, Feedback, Recognition) との連携**:
    - **Conversation**: 1on1などでの質の高い対話。
    - **Feedback**: リアルタイムでの具体的なフィードバック。
    - **Recognition**: 成果や貢献に対するタイムリーな承認。
    - これらを通じて、OKR振り返りを形骸化させない文化を醸成します。
- **自信度の活用**: OKR設定時に「達成自信度」を設定し、振り返り時にその予測と結果を比較分析することで、目標設定の精度を高めることができます。
- **心理的安全性**: 達成率が低くても非難されることなく、失敗から学ぶことを奨励する文化が不可欠です。

## 振り返りのタイミング

- **週次ミーティング (Weekly Check-in)**: 短期的な進捗確認、課題の早期発見と解決。
- **中間レビュー (Mid-term Review)**: 必要に応じた軌道修正やリソースの再配分。
- **期末レビュー (Quarterly/Final Review)**: 総合的な評価、学習の抽出、および次期OKRへのインプット。

## Related Concepts

- **PDCAサイクル** - a continuous improvement framework
    - OKR振り返りは、PDCAにおけるCheck（評価）とAct（改善）のフェーズを具体的に実践する手法です。
- **AAR (After Action Review)** - a structured review process
    - AARが軍事演習など単一のイベント後に用いられるのに対し、OKR振り返りはビジネスの四半期サイクルなど定期的なリズムに組み込まれます。
- **KPT法 (Keep/Problem/Try)** - a simple retrospective technique
    - OKR振り返りの「課題と学びの分析」「次期アクションへの反映」フェーズで活用できる具体的なフレームワークの一つです。
- **ストレッチゴール** - ambitious goal-setting
    - OKRは、現状維持ではなく挑戦的な目標（ストレッチゴール）を推奨しており、振り返りもその達成度を前提に行われます。

# 応用例

## ソフトウェア開発チーム
- **Objective**: ユーザーエンゲージメントを劇的に向上させる
- **KR**: 1日あたりのアクティブユーザー数を20%増加させる → **結果: 12%増 (スコア: 0.6)**
- **振り返り**:
    - **成果**: 新機能Aの利用率が非常に高く、エンゲージメント向上に貢献した。
    - **課題**: 技術的負債の返済に予想以上の時間がかかり、予定していた機能Bのリリースが遅れた。
    - **学び**: ユーザーインタビューで得られた「UIの複雑さ」というフィードバックが、エンゲージメント停滞の一因かもしれない。
    - **次期アクション**: 次期OKRでは「UI/UXの抜本的改善」を主要なObjectiveの一つに設定する。

## 個人のキャリア開発
- **Objective**: データ分析の専門家としてチームに貢献する
- **KR**: 関連書籍を3冊読了し、社内勉強会で発表する → **結果: 3冊読了、発表済み (スコア: 1.0)**
- **KR**: 実務プロジェクトで分析モデルを2つ構築する → **結果: 1つ構築 (スコア: 0.5)**
- **振り返り**:
    - **成果**: インプットは計画通り進んだ。勉強会での発表は高評価だった。
    - **課題**: 2つ目のモデル構築に着手する時間が、通常業務の多忙さから確保できなかった。
    - **学び**: スキル習得（インプット）と実務応用（アウトプット）のバランスを取るには、意図的な時間確保が必要。
    - **次期アクション**: マネージャーと相談し、週の20%を分析業務に充てる時間を確保する。
