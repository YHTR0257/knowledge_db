# Key Points

- やったこと、わかったこと、次にやることで振り返りを行う方法
- 行動、気づき、そして次の計画を明確に連携させることができる

# Context
- 定期的な振り返りを行うことにより、自分の現在地を知ることができる
- 過程を追うことでその過程が正しいものだったかを検証することができる
- 検証結果をもとに、次に取るべきアクションを知ることができる

# Body
YWTメソッドは、以下の3つの要素で構成されるシンプルな振り返りのフレームワークです。

### Y: やったこと (What you did)
- 具体的にどのような行動をとったか、事実を客観的に書き出す。
- 良かった点、悪かった点を含めて、実際に行ったことをリストアップする。
- **例:** 「Aというタスクを完了させた」「Bの会議に参加した」

### W: わかったこと (What you learned)
- 「やったこと」を通じて、どのような学びや気づきがあったかを記述する。
- 成功した要因、失敗した原因、改善点、新しい知識などを言語化する。
- **例:** 「Aのタスクはxxのツールを使うと効率化できることがわかった」「Bの会議では、事前の情報共有が重要だと感じた」

### T: 次にやること (What you will do next)
- 「わかったこと」を基に、次に取り組むべき具体的なアクションプランを立てる。
- 抽象的な目標ではなく、具体的で実行可能なタスクに落とし込むことが重要。
- **例:** 「次の類似タスクではxxのツールを導入する」「次回の会議前にアジェンダと資料を共有する」

この3つのステップを繰り返すことで、経験を学びに、そして学びを次の行動へと着実に繋げることができます。


## Counterarguments
- 形骸化しやすく、表面的な振り返りに終わる可能性がある。
- 次にやることに繋がらない場合、単なる記録で終わってしまう。
- 定期的な実施が負担になる可能性がある。

## Applications
- 個人の日次・週次・月次の振り返り
- プロジェクトのKPT (Keep, Problem, Try) と同様にチームでの振り返り
- 目標達成プロセスの進捗確認と軌道修正

## Related Concepts

関係する概念とその関係性について1行で簡潔に記載する

- KPT (Keep, Problem, Try) - 類似
    - 「やったこと」が「Keep」と「Problem」に、「次にやること」が「Try」に相当し、チームでの振り返りによく用いられる。
- PDCAサイクル - 類似
    - 計画 (Plan) → 実行 (Do) → 評価 (Check) → 改善 (Act) のサイクルを回すことで継続的な改善を促すフレームワーク。YWTはPDCAの「Do」と「Check」の部分に密接に関連する。

# 応用例

### 個人の日報・週報
毎日の業務の終わりに「やったこと」「わかったこと」「次にやること」を記録することで、日々の学びを最大化し、次の行動を明確にする。

### チームのスプリントレビュー
スプリントの終わりにチームメンバーでYWTを共有し、スプリントの成果、課題、次のスプリントでの改善点を洗い出す。

### プロジェクトのフェーズ終了時レビュー
プロジェクトの各フェーズが終了した際にYWTを用いて振り返りを行い、プロジェクト全体の進行を最適化する。
