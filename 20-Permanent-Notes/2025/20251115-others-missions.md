# 色々な人のミッションまとめ

このドキュメントでは、さまざまな人々のミッションをまとめています。各人の専門性や目標に基づいて、どのようにミッションを達成しようとしているかを示しています。
ミッションの中身、ミッションを達成するための手段、専門性などを整理しています。

## Missionとは

Missionとは、その人が人生で果たしたい・取り組むべきと思っていることについてを言語化したものです。
その人の原動力となるため、キャリアにおいて重要な役割を果たします。

### 概念

- MotivationとMissionは区別しなければなりません。
- **誰かに価値を提供することがMissionの本質**です。提供するものに注目すると以下のように分類できます。
	- 誰かのためになること、誰かの助けになることを実現して初めてMissionだと言えると結論づけます。

|                | **有形リソース中心** | **無形リソース中心** |
| -------------- | ------------ | ------------ |
| **既存価値の移転・提供** | リソース提供型      | 知識・情報伝達型     |
| **新規価値の創造**    | 創造・開発型       | 仲介・調整型       |

1.  **リソース提供型**
    組織やプロジェクトに必要なリソースを供給する
    *   労働力として、手を集める
    *   モノを運ぶ
    *   モノを仕入れる
    *   専門技能を提供する
    *   基礎技術を提供する

2.  **知識・情報伝達型**
    既存の知識や情報を必要な人に届ける
    *   知られていない物を教える
    *   情報を提供する
    *   知見を提供する
    *   売るための情報を提供する
        *   既存のマーケットを分析する

3.  **創造・開発型**
    新しい価値やソリューションを生み出す
    *   世の中にないものを作る
    *   より売りやすいモノを作る
    *   埋もれている技術を現在見つかっている課題に対して実装する

4.  **仲介・調整型**
    需要と供給、人と人をつなぐ
    *   組織を動かす
    *   誰かの代わりに仕事をあつめる
    *   モノを売る
    *   共感してくれる人を集める
    *   世界を見せる
    *   物を売る
    *   物を買って紹介する

## 基本的な構成

### What he / she is doing
- その人が現在取り組んでいること、専門性

### Mission
- ミッションの概要
- 目的や目標

### How
- ミッションを達成するための具体的な手段やアプローチ

### What he / she is doing

- 個人のミッションはなく、会社全体のミッションに従う形
- どっちかというと、「面白いこと、技術を深める」ということがモチベーションとなっている

- 会社のミッションに従い、プロジェクトマネジメントを行う

- 再現性が高いという意味でプログラムやシステムに魅力を感じている

    - LMにいるのは、その辺りを技術的に深めていける環境があるから。企業としてもミッションは大きく変わらないと思っているから残っている。
