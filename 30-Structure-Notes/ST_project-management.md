# Project Management

## Overview

プロジェクトマネジメントの体系的な知識とフレームワークを、PDCAサイクルの各段階に沿って整理したストラクチャーノート。目標設定、計画立案、実行管理、評価、改善という一連のプロセスで使用する各種手法と、それらを段階的に習得するための学習パスを提供する。

個人のタスク管理から組織レベルのプロジェクト運営まで、幅広いスケールで応用可能な原則とツールを網羅している。

## プロジェクト管理の全体像

プロジェクトマネジメントにおいて、PDCAサイクルは継続的な改善と学習の基盤となる。各フェーズで適切なフレームワークを活用することで、目標達成の精度と効率を高めることができる。

## Concept Hierarchy

### Level 1: 基礎的な概念（個人レベル）

初めてプロジェクトマネジメントを学ぶ人向けの、シンプルで実践しやすい基本的なフレームワーク。

- **目標設定の基本: SMART Framework** - 具体的で測定可能な目標を設定する基礎
    - [20260102_smart-framework.md](../20-Permanent-Notes/2026/20260102_smart-framework.md) - 5つの要素（Specific, Measurable, Achievable, Relevant, Time-bound）による目標設定の原則
    - 個人のタスクやプロジェクトで即座に活用可能

- **日常的な振り返り: YWT Method** - 最もシンプルな振り返りフレームワーク
    - [20260102_ywt-method.md](../20-Permanent-Notes/2026/20260102_ywt-method.md) - やったこと・わかったこと・次にやることの3ステップ
    - 毎日5分で実践できる継続的な学習サイクル

- **タスクの実行管理: Task Management Skills** - プロジェクト実行の基盤スキル
    - [20240910-task-management-skills.md](../20-Permanent-Notes/2024/20240910-task-management-skills.md) - 優先順位設定、進捗管理、報連相の実践
    - 複数のタスクを効率的に管理する能力

### Level 2: 具体的な応用（チーム・プロジェクトレベル）

複数人でのプロジェクト運営や、より構造化された管理手法を習得する段階。

- **チームでの振り返り: 4L Method & AAR** - チームの学習と改善を促進
    - [20260103_4l-method.md](../20-Permanent-Notes/2026/20260103_4l-method.md) - Liked/Learned/Lacked/Longed forによるスプリント振り返り
    - [20260102_aar-method.md](../20-Permanent-Notes/2026/20260102_aar-method.md) - 計画・実績・原因・改善の厳密な分析フレームワーク
    - 心理的安全性の確保と深い原因分析のバランス

- **目標管理の実践: OKR Review** - 組織と個人の目標を連携
    - [20260103_okr-review.md](../20-Permanent-Notes/2026/20260103_okr-review.md) - Objectivesと Key Resultsによる野心的目標の設定と評価
    - スコアリング、学びの分析、次期アクションへの反映

- **リスクの評価と対応: Risk Assessment** - 不確実性への備え
    - [20241201-risk-assessment-frameworks.md](../20-Permanent-Notes/2024/20241201-risk-assessment-frameworks.md) - リスク特定、評価、対応戦略の選択
    - プロジェクトの失敗確率を低減する予防的アプローチ

- **複数基準での意思決定: Multi-Criteria Decision Analysis** - 複雑な選択肢の評価
    - [20241201-multi-criteria-decision-analysis.md](../20-Permanent-Notes/2024/20241201-multi-criteria-decision-analysis.md) - 複数の評価軸を統合した意思決定手法
    - トレードオフを明確化し、最適な選択を支援

- **能力の可視化: Skill Evaluation Framework** - 成長の進捗を測定
    - [20240910-skill-evaluation-framework.md](../20-Permanent-Notes/2024/20240910-skill-evaluation-framework.md) - 技術・ドメイン・ソフトスキルの体系的評価
    - プロジェクトを通じた学習効果の測定

### Level 3: 高度なトピック（組織・戦略レベル）

組織全体の戦略的マネジメントや、複数のフレームワークを統合した高度な運用。

- **戦略的評価: Balanced Scorecard** - 多角的な組織評価
    - [20260103_balanced-scorecard.md](../20-Permanent-Notes/2026/20260103_balanced-scorecard.md) - 財務・顧客・業務プロセス・学習と成長の4つの視点
    - 長期的な企業価値向上と戦略実行の可視化

- **振り返りフレームワークの統合活用: Review Frameworks** - 状況に応じた最適な手法選択
    - [20260102_review-frameworks.md](../20-Permanent-Notes/2026/20260102_review-frameworks.md) - AAR、YWT、OKR、BSC、4Lなどの比較と使い分け
    - チームの成熟度、目的、時間的制約に応じたフレームワーク選択
    - 複数フレームワークの組み合わせによる相乗効果

- **ミッションと戦略の整合: Mission Statement** - プロジェクトの存在意義
    - [20260103_mission-statement.md](../20-Permanent-Notes/2026/20260103_mission-statement.md) - ビジョン、ミッション、バリューの明確化
    - 組織やプロジェクトの方向性を統一し、意思決定の指針とする

## Key Relationships

### PDCAサイクルの循環性

各フェーズは独立して機能するのではなく、継続的なサイクルとして連携する。

- **Plan → Do**: SMART目標やOKRで設定した目標を、タスク管理スキルで実行可能な単位に分解
- **Do → Check**: 実行中のYWTやタスク管理から得られたデータを、4LやAARで体系的に評価
- **Check → Act**: 振り返りから得た学びを、次の計画サイクルに反映（OKR Reviewの次期OKR設定など）
- **Act → Plan**: 改善策を新しい目標設定に組み込み、より高度な目標へ挑戦

### 階層的な目標設定

プロジェクトマネジメントでは、異なるレベルの目標が階層的に連携する。

- **戦略レベル**: Mission Statement、Balanced Scorecard（組織全体の方向性）
- **戦術レベル**: OKR、SMART目標（四半期・年次の具体的目標）
- **実行レベル**: タスク管理、YWT（日次・週次の行動）

上位の目標が下位の目標の文脈を提供し、下位の実行が上位の目標達成に貢献する。

### 振り返りの深度とタイミング

振り返りフレームワークは、目的と状況によって使い分ける。

- **日常的な振り返り（5-15分）**: YWT - 個人の学習サイクル
- **短期サイクル（30-60分）**: 4L、KPT - スプリント・週次の改善
- **中期サイクル（1-2時間）**: AAR、OKR Review - 四半期・プロジェクト完了時
- **長期サイクル（2-4時間）**: Balanced Scorecard - 年次の戦略評価

タイミングと深度を適切に設定することで、形骸化を防ぎ、継続的な改善を実現する。

## Learning Path

プロジェクトマネジメントの能力を段階的に高めるための推奨学習順序。

### 1. 入門フェーズ（1-2ヶ月）

まず個人レベルで基本的なフレームワークを実践し、習慣化する。

**学習順序**:
1. **[SMART Framework](../20-Permanent-Notes/2026/20260102_smart-framework.md)** を学び、個人の目標を設定する（1週目）
2. **[YWT Method](../20-Permanent-Notes/2026/20260102_ywt-method.md)** を使って毎日5分の振り返りを習慣化する（2週目〜継続）
3. **[Task Management Skills](../20-Permanent-Notes/2024/20240910-task-management-skills.md)** で複数タスクの管理手法を学ぶ（3-4週目）

**マイルストーン**: 1ヶ月間、毎日YWTを記録し、週次でSMART目標の進捗を評価できる

### 2. 実践フェーズ（3-6ヶ月）

チームやプロジェクトで実際に活用し、より構造化された手法を導入する。

**学習順序**:
1. **[AAR Method](../20-Permanent-Notes/2026/20260102_aar-method.md)** でプロジェクト完了後の深い振り返りを実施（月次）
2. **[4L Method](../20-Permanent-Notes/2026/20260103_4l-method.md)** をチームのスプリント振り返りに導入（隔週）
3. **[OKR Review](../20-Permanent-Notes/2026/20260103_okr-review.md)** で四半期目標を設定・評価する（四半期ごと）
4. **[Risk Assessment](../20-Permanent-Notes/2024/20241201-risk-assessment-frameworks.md)** で計画段階でのリスク管理を実践

**マイルストーン**: 四半期OKRを設定し、チームで定期的な振り返りを実施できる

### 3. 統合フェーズ（6ヶ月〜）

複数のフレームワークを組み合わせ、状況に応じた最適な手法を選択できるようになる。

**学習順序**:
1. **[Review Frameworks](../20-Permanent-Notes/2026/20260102_review-frameworks.md)** で各手法の使い分けを学ぶ
2. **[Skill Evaluation Framework](../20-Permanent-Notes/2024/20240910-skill-evaluation-framework.md)** で自己の成長を定期評価
3. **[Balanced Scorecard](../20-Permanent-Notes/2026/20260103_balanced-scorecard.md)** で組織・プロジェクトの多角的評価を実践
4. **[Mission Statement](../20-Permanent-Notes/2026/20260103_mission-statement.md)** でプロジェクトの方向性を明確化

**マイルストーン**: プロジェクトの性質とチームの成熟度に応じて、適切なフレームワークを選択・カスタマイズできる

### 4. マスタリーフェーズ（継続的）

組織全体の仕組みとして定着させ、継続的な改善文化を醸成する。

- 複数プロジェクトを横断した学習の共有
- フレームワークの組織カスタマイズ
- メンタリングを通じた知識伝承
- 新しい手法の実験と評価

## Practical Examples

### 例1: スタートアップでの段階的導入

**フェーズ1: 創業期（〜10名、最初の3ヶ月）**
- **Plan**: 創業者が[Mission Statement](../20-Permanent-Notes/2026/20260103_mission-statement.md)を策定、初期メンバーと共有
- **Do**: 各メンバーが[SMART Framework](../20-Permanent-Notes/2026/20260102_smart-framework.md)で個人目標を設定、[YWT](../20-Permanent-Notes/2026/20260102_ywt-method.md)で日次振り返り
- **Check**: 週次で[4L Method](../20-Permanent-Notes/2026/20260103_4l-method.md)を全員で実施（30分）
- **Act**: 月次で[AAR](../20-Permanent-Notes/2026/20260102_aar-method.md)を実施し、翌月の優先事項を決定（1時間）

**フェーズ2: 拡大期（〜50名、6ヶ月目〜）**
- **Plan**: 部門ごとに[OKR](../20-Permanent-Notes/2026/20260103_okr-review.md)を設定し、全社目標と連携
- **Do**: スクラムチームごとに2週間スプリントを運営
- **Check**: スプリントごとに[4L](../20-Permanent-Notes/2026/20260103_4l-method.md)、四半期ごとに[OKR Review](../20-Permanent-Notes/2026/20260103_okr-review.md)
- **Act**: 四半期の振り返りから次のOKRを設定、[Review Frameworks](../20-Permanent-Notes/2026/20260102_review-frameworks.md)を参照して手法を最適化

**フェーズ3: 成長期（50名〜、1年目〜）**
- **Plan**: 経営層が[Balanced Scorecard](../20-Permanent-Notes/2026/20260103_balanced-scorecard.md)で戦略を多角的に設計
- **Check**: 半期ごとにBSCで組織全体の健全性を評価
- **Act**: 全社レトロスペクティブで[Review Frameworks](../20-Permanent-Notes/2026/20260102_review-frameworks.md)の使い分けを最適化

### 例2: 個人のキャリア開発プロジェクト

**目標**: 「データサイエンティストとして転職し、年収を20%向上させる」（12ヶ月計画）

**Plan（1ヶ月目）**:
- [SMART Framework](../20-Permanent-Notes/2026/20260102_smart-framework.md)で具体的目標を設定
  - S: 機械学習エンジニア職に転職
  - M: 年収600万円→720万円、3つのポートフォリオプロジェクト完成
  - A: 現在の基礎知識+6ヶ月の集中学習で達成可能
  - R: キャリアビジョン「データ駆動の意思決定を支援する専門家」に合致
  - T: 2026年12月末までに内定獲得
- [Risk Assessment](../20-Permanent-Notes/2024/20241201-risk-assessment-frameworks.md)でリスク洗い出し
  - スキル不足リスク → 毎週10時間の学習時間確保
  - 収入減リスク → 現職を続けながら転職活動
- 四半期OKRを設定
  - Q1: Kaggleでメダル1つ獲得、SQL・Pythonの基礎を固める
  - Q2: 実務に近いプロジェクト2つ完成、技術ブログ5記事執筆
  - Q3: 企業10社にエントリー、面接5社通過
  - Q4: 内定獲得、入社準備

**Do（2-11ヶ月目）**:
- [Task Management](../20-Permanent-Notes/2024/20240910-task-management-skills.md)で週次・月次計画を立案
  - 毎週: Kaggle 5時間、Coursera 3時間、ポートフォリオ開発 2時間
  - 月次: ブログ執筆、スキルの可視化
- [YWT Method](../20-Permanent-Notes/2026/20260102_ywt-method.md)で毎日振り返り（5分）
  - やったこと: 「Kaggleのタイタニック問題でランダムフォレストを実装」
  - わかったこと: 「特徴量エンジニアリングが精度に大きく影響する」
  - 次にやること: 「明日は欠損値処理の手法を3つ試す」

**Check（四半期ごと）**:
- [OKR Review](../20-Permanent-Notes/2026/20260103_okr-review.md)でKRのスコアリング
  - Q1結果: Kaggleメダル0個（スコア0.0）、SQL・Python習得（スコア1.0）
  - 成功要因: Courseraの構造化されたカリキュラムが効果的だった
  - 阻害要因: Kaggleコンペの難易度を見誤った、時間配分が不適切
- [AAR](../20-Permanent-Notes/2026/20260102_aar-method.md)で深い分析（月次）
  - 計画: 週10時間の学習
  - 実績: 平均7.5時間（75%達成）
  - 原因: 仕事の繁忙期と重なり、平日の学習時間が確保できなかった
  - 改善策: 早朝学習に切り替え（朝6-7時を確保）、週末に集中時間を設定

**Act（各振り返り後）**:
- Q1の振り返りから次のアクションを決定
  - Kaggleは初心者向けコンペに絞る（難易度調整）
  - 朝型学習にシフト、カレンダーにブロック予約
  - ポートフォリオ開発に重点をシフト（実務スキルの証明が重要）
- [Skill Evaluation Framework](../20-Permanent-Notes/2024/20240910-skill-evaluation-framework.md)で進捗を可視化（四半期ごと）
  - 技術スキル: Python 3→4、機械学習 2→3.5
  - ドメイン知識: データ分析 3→4
  - ソフトスキル: プレゼン 3→3.5（技術ブログでの説明力向上）

**最終結果（12ヶ月目）**:
- 目標達成: 3社から内定、年収700万円で入社決定（目標720万円の97%）
- 学び: ポートフォリオの質が面接突破の鍵だった、ネットワーキングの重要性を過小評価していた
- 次のサイクル: 入社後の[OKR](../20-Permanent-Notes/2026/20260103_okr-review.md)を設定し、新しいPDCAを開始

### 例3: 研究プロジェクトでのフレームワーク活用

**プロジェクト**: 「新素材の機械学習予測モデル開発」（6ヶ月、チーム5名）

**Plan（初月）**:
- [Mission Statement](../20-Permanent-Notes/2026/20260103_mission-statement.md)を策定
  - ミッション: 「データ駆動型材料設計により、開発期間を50%短縮する」
  - ビジョン: 「材料科学とAIの融合により、持続可能な社会に貢献」
- [SMART + OKR](../20-Permanent-Notes/2026/20260102_smart-framework.md)で目標設定
  - Objective: 高精度な予測モデルを構築し、論文を国際会議で発表
  - KR1: 予測精度R²=0.85以上達成
  - KR2: 実験データ1000サンプル収集
  - KR3: 国際会議Aに採択（投稿締切5ヶ月後）
- [Risk Assessment](../20-Permanent-Notes/2024/20241201-risk-assessment-frameworks.md)
  - 実験装置の故障リスク → バックアップ装置の確保
